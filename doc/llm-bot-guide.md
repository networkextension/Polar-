# LLM Config 和 Bot 使用指南

这是一份给普通用户的小白说明，目标是让你在站内自己配置模型，并创建一个能聊天的 Bot。

如果你不懂接口协议、线程、上下文这些技术词，也没关系，照着下面一步一步填就能用。

## 1. 先理解两个东西

在设置中心里，你会看到两块：

- `LLM Config`
- `Bot 管理`

它们的区别很简单：

- `LLM Config`
  - 负责连接模型
  - 主要填写 `Base URL`、`Model`、`API Key`
- `Bot`
  - 负责 Bot 自己的名字、简介、说话风格
  - 主要填写 `Bot 名称`、`简介`、`Prompt`
  - 并选择“默认用哪个 LLM Config”

可以把它理解成：

- `LLM Config` = 车子的发动机
- `Bot` = 司机的人设

## 2. 最简单的使用流程

推荐按这个顺序操作：

1. 先创建一个 `LLM Config`
2. 用“测试配置”确认这个模型能连通
3. 再去 `Bot 管理` 创建一个 Bot
4. 给 Bot 发私信开始聊天

## 3. 如何填写 LLM Config

进入设置中心的 `设置` 页签，找到 `LLM Config`。

你通常需要填写这些字段：

- `名称`
  - 这是你自己看的名字
  - 例如：`OpenAI`、`Qwen 测试`、`Kimi 生产`
- `Base URL`
  - 模型接口地址
  - 这个必须是服务商要求的完整接口地址
- `Model`
  - 模型名
  - 必须和服务商实际支持的名字完全一致
- `API Key`
  - 你的模型访问密钥
- `测试 Prompt（可选）`
  - 只用于测试连通性
  - 不会成为 Bot 的长期人设

### 3.1 一个可参考的例子

如果你用的是 OpenAI Chat Completions 兼容接口，通常会像这样：

- `名称`：`OpenAI`
- `Base URL`：`https://api.openai.com/v1/chat/completions`
- `Model`：`gpt-4.1-mini`
- `API Key`：你的 key

如果你用的是其他兼容 OpenAI 的服务，也通常是类似格式，但具体以服务商文档为准。

如果你用的是 Gemini 原生接口，推荐这样填：

- `名称`：`Gemini`
- `Base URL`：`https://generativelanguage.googleapis.com/v1beta`
- `Model`：`gemini-2.5-flash`
- `API Key`：你的 Gemini API key

注意：

- 不需要把 `?key=...` 拼进 `Base URL`
- 也不需要手动把完整 `:generateContent` 路径写死
- 服务端会识别 Google Gemini 地址，并自动按 Gemini 的原生请求格式发送

如果你已经习惯用 curl，它最终对应的大致效果会是：

```bash
curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key=$API_KEY" \
  -H 'Content-Type: application/json' \
  -X POST \
  -d '{
    "systemInstruction": {
      "parts": [{"text": "你的 Bot Prompt"}]
    },
    "contents": [
      {"role": "user", "parts": [{"text": "用户问题"}]}
    ]
  }'
```

## 4. 为什么一定要先点“测试配置”

很多人不是模型不能用，而是填错了其中一个字段：

- `Base URL` 少了路径
- `Model` 名字写错
- `API Key` 复制多了空格
- 接口和账号权限不匹配

所以推荐每次保存前都先点一次：

- `测试配置`

测试结果会直接显示在 `LLM Config` 标题右侧。

常见结果：

- 显示成功
  - 说明这套配置可以连接模型
- 显示失败
  - 一般会直接带服务端返回的错误

## 5. 测试失败时怎么排查

最常见的问题就这几类：

- `model not found`
  - 模型名写错了
  - 或者你的账号没有这个模型权限
- `url.not_found`
  - `Base URL` 不完整
  - 很多服务不能只填域名或 `/v1`
- `permission denied`
  - key 没权限
- 超时或网络错误
  - 服务商接口不稳定
  - 或地址不可访问

一个很实用的排查习惯：

- 先只保留最基本配置
- 不要同时改很多项
- 每次只改一个字段，再测试一次

## 6. 如何创建 Bot

当 `LLM Config` 测试通过后，进入设置中心的 `Bot 管理`。

创建 Bot 时，你通常会看到这些内容：

- `Bot 名称`
  - 例如：`翻译助理`、`产品经理助手`、`代码解释器`
- `简介`
  - 简短描述这个 Bot 是干什么的
- `默认 LLM Config`
  - 选择刚才测试通过的那套模型配置
- `Bot Prompt`
  - 这是 Bot 的核心设定
  - 决定它说话风格、角色和做事方式

## 7. Bot Prompt 应该怎么写

你可以把 `Bot Prompt` 理解成：

- 这个 Bot 是谁
- 它应该怎样回答
- 它要重点帮助我做什么

### 7.1 一个简单例子

如果你想做一个翻译 Bot，可以写：

```text
你是一个专业中英双语翻译助手。
默认把用户输入翻译成更自然、简洁、商务风格的英文。
如果用户明确要求中文，就输出中文。
不要解释太多，优先直接给结果。
```

### 7.2 再给一个例子

如果你想做一个写作润色 Bot，可以写：

```text
你是一个中文写作润色助手。
请把用户输入改得更清晰、更自然、更有条理。
默认保持原意，不随意扩写。
如果内容像标题、摘要、要点列表，就按更适合阅读的格式输出。
```

### 7.3 Prompt 的建议

- 尽量写清楚角色
- 尽量写清楚输出风格
- 尽量写清楚默认行为
- 不要一上来写得特别长

先写一个简单可用版，聊几轮后再慢慢调，通常效果更好。

## 8. LLM Config 和 Bot Prompt 的分工

这是最容易搞混的地方。

你只要记住：

- `LLM Config`
  - 决定连哪个模型
- `Bot Prompt`
  - 决定 Bot 的性格和回答方式

也就是说：

- 换模型，不等于换人设
- 改 Prompt，不等于换模型

## 9. 聊天时的“话题”和“切换模型”

现在一个 Bot 私聊里可以有多个话题。

你可以理解成：

- 同一个 Bot
- 但每个话题都是一段相对独立的上下文

例如：

- 话题 A：合同翻译
- 话题 B：周报润色
- 话题 C：生成招聘文案

这几个话题不会完全混在一起。

### 9.1 切换模型是怎么生效的

聊天页里切换模型时，切换的是：

- 当前话题使用的 `LLM Config`

不会变的是：

- Bot 名称
- Bot 简介
- Bot Prompt

所以你可以把它理解成：

- Bot 还是那个 Bot
- 只是这次话题换了一个“大脑”

### 9.2 切换后会影响什么

- 只影响后续消息
- 不会重跑历史消息

## 10. 为什么有时会看到失败消息和 Retry

有些模型服务会偶发失败，比如：

- 临时超时
- 权限波动
- 服务商接口异常

这时聊天里可能会出现一条失败消息，并带：

- `Retry`

它的作用是：

- 直接拿你上一条用户消息重新请求一次模型
- 你不用自己再复制粘贴重发

如果重试成功，原来的失败消息会自动从聊天里移除。

## 11. `system` 和你自己创建的 Bot 有什么区别

系统里有一个内置的：

- `system`

它是官方默认 AI 助理。

区别是：

- 给 `system` 发消息时
  - 系统会读取程序运行目录下的文档摘要
- 给你自己创建的 Bot 发消息时
  - 默认不会读取这些运行目录文档
  - 主要只看当前聊天话题上下文 + 你的 Bot Prompt

所以如果你想做自己专属的助手，推荐直接创建自己的 Bot。

## 12. 新手最推荐的配置方式

如果你第一次用，建议这样开始：

1. 先建一个最基础的 `LLM Config`
2. 名称写简单一点，比如 `我的主模型`
3. 先点“测试配置”
4. 再创建一个 Bot
5. Prompt 只写 3 到 5 行
6. 先聊几轮看看效果
7. 再慢慢调整 Prompt

不要一开始就：

- 建很多个 Config
- 写很长很复杂的 Prompt
- 同时测很多模型

这样更容易把自己绕晕。

## 13. 推荐的起步模板

### 13.1 通用助手

```text
你是一个清晰、稳定、直接的中文助理。
优先给出可执行答案。
如果用户的问题不够明确，请先用一句话指出缺失信息。
回答尽量简洁，不要空话。
```

### 13.2 翻译助手

```text
你是一个专业翻译助手。
默认把用户输入翻译成自然、准确、简洁的目标语言。
如果用户没有指定语言，就根据上下文做最合理判断。
优先给结果，少做解释。
```

### 13.3 写作助手

```text
你是一个写作优化助手。
请在不改变原意的前提下，优化语句的清晰度、逻辑和可读性。
如果适合列表表达，就整理成列表。
默认输出中文。
```

## 14. 常见问题

### 14.1 为什么测试成功了，但聊天效果一般

因为：

- 测试配置只说明“能连上”
- 不代表 Prompt 已经调好

这时候优先改：

- `Bot Prompt`

### 14.2 为什么同一个 Bot 不同话题表现不一样

因为：

- 不同话题上下文不同
- 当前话题可能切换了不同模型

### 14.3 为什么换模型后历史消息没变

这是正常的。

当前设计是：

- 切换模型只影响后续回复
- 不会重做历史消息

## 15. 一句话总结

如果你只想快速用起来，就记住这三句：

- 先配 `LLM Config`
- 一定先点“测试配置”
- 再创建 Bot，并用 `Bot Prompt` 决定它的风格

这样通常 5 分钟内就能把第一个 Bot 跑起来。
